so what's going on behind the scenes?

We're using/leveraging/extending/working with:

  • TheDataWeb metadata repository
  • AFF presentation metadata
  • The Content Management System for unstructured content metadata
  • External best practices and standards - DDI/schema.org/DOIs
  • Geography Division to develop standards for documenting geographic feature and FIPS code metadata

Our goal with this is to:

  • Maintain the integrity of the data over time and across data sets
  • Maximize performance
  • Enable dynamic standardization of disparate data types across data sets
  • Establish two-way integration between supported data sets
  • Increase visibility across data sets
  • Identify and reduce inconsistencies across data sets supported by the metadata over time
  • Increase internal awareness of the landscape of data supported by the metadata
  • Ensure that the integration of Census public-use data is complete, consistent, and correct
